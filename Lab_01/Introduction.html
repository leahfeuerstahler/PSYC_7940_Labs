Compile to PDF or Word

  • Quarto
  • R Markdown

Use Quarto if it works on your computer.

  1. Open a file: File \(\rightarrow\) New File \(\rightarrow\) Quarto Document

  2. Select “Document” in the leftmost panel.

  3. Select “PDF” or “Word” as the output format and click “Create”.

  4. A template Quarto document will be opened in RStudio. Click “Render” or File \(\rightarrow\) Render Document. Save the file first.

  5. If successful, a Word or PDF file should appear in the same folder in which you saved the file.

R Markdown is older than Quarto. Only use this option if Quarto gives errors (e.g., Quarto files don’t render on Windows computers with ARM processors).

  1. Open a new file: File \(\rightarrow\) New File \(\rightarrow\) R Markdown

  2. Select “Document” in the leftmost panel.

  3. Select “PDF” or “Word” as the output format and click “OK”.

  4. A template R Markdown document will be opened in RStudio. Click “Knit” or File \(\rightarrow\) Knit Document. Save the file first.

  5. If successful, a Word or PDF file should appear in the same folder in which you saved the file.

Introduction to ggplot2

Plotting is an essential component in this class. We will use

  • ggplot2 is a R package which is part of the tidyverse.

  • If you have already installed tidyverse, you can use ggplot2.

# install.packages("tidyverse")
library(tidyverse)
  • In this lab, we will focus on plotting with ggplot2. Even though it is not always the best solution, it is well-integrated with the other packages that we will us for Bayesian statistics and it is user-friendly. If you prefer, you may instead use basic R tools to make plots in the assignments.

stan and brms

  • stan is a command-line program for statistical modeling that is currently the best for general-purpose Bayesian computation.

  • Most users interact with stan through integrations with programming languages like R or python.

  • In this class, we will primarily interact with stan through the brms package in R. A major advantage of brms is that its syntax is similar to lm().

  • For example:

# Ordinary regression:
lm(outcome ~ predictor1 + predictor2)

# Bayesian regression with brms:
brm(outcome ~ predictor1 + predictor2)

brms calls another R integration to stan: rstan.

rstan
